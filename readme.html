Specify all the different route your app supports:

For a new user:
1. Go to /login.html, then sign up and return to /login.html
2. Log in to the system via the button, you will be automatically transferd to the /store.html.
3. Purchase in the store via 'add to cart' button near the product, change the quantity and remove products as you like.
4. When you done press the 'PURCHASE' button in the end of the page.
5. Whenever you like you can press the 'about' in the upper bar in order to read more about the company.
6. Whenever you like you can press the 'contcat us' in order to keep in touch with the team.
7. If you are logged in as the admin (user: admin, password: admin. found in the system by default). go to the 'Admin Interface' by clicking on the upper bar in the store. /index.html will be openned.
8. In the Admin Interface click on 'Load Log' and see all users purchases.
9. In any time you can logout from the system by pressing on Logout in the upper bar.

How did you make your store secured?

If a user is not logged in he cannot reach the store, and he is redirected to the login page.
Only a user that is logged in can purchase in the store.
Any logged in user is provided with a cookie. With the cookie a user can retrive its username, but cannot recieve the password which is stored in a diffrent DB for security reasons.
Only with the sysAdmin user can aproach the Admin Interface via the store through the link.
The admin can't load the log in admin interface without a cookie.
If somehow a user without permissions trys to load the log, he will be blocked by another security wall based on the provided cookie, which will not load the log, and user will be redirected to login page after alert of appropriate message.
The sysAdmin user is found in the system by default, and no one can sign in a new user with the same name since the 'sign in' mechanism does not allow it.
